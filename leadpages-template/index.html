MODIFIED GETTING STARTED

The file and folder structure of a LeadPages™ template is relatively simple. There are a few gotchas and best-practices worth noting but fundamentally a template is comprised of HTML, CSS, Javascript, and some media assets. Skip to the About section to skip right to a sample template.

Template File and Folder Structure

leadpages-template/
|--- css/
|    |---- style.css
|--- img/
|    |---- image1.jpg
|    |---- image2.png
|    |---- image3.gif
|    |---- ...
|--- index.html
|--- js/
|    |---- functions.js
|    |---- html5shiv.js
|    |---- jquery-1.9.1.min.js
|--- meta/
|    |---- form.html
|    |---- template.json

Recommendations & Notes About Files and Folders

  • Required! Every template must have a leadpages-template folder at its root.
  • Stylesheets must be saved locally and included with the template.
  • Images included in the img/ directory that are not used within the template will cause errors when the template is uploaded.
  • Required! HTML5Shiv must be included and linked within index.html.
  • Required! The meta/ directory must exist and contain template.json and form.html or the template will fail to upload. form.html can be empty.
  • Required! <meta charset="utf-8" /> must exist in the <head/> of index.html.
  • Best practice! Use web-optimized images. A tool like Smush.it can be quite handy.
  • Best practice! Put common or custom Javascript code in a single file in the js/ directory (i.e. js/functions.js).

Metadata/Instructions

The instructions for the LeadPages editor. Herein lies the magic.

"meta/template.json" Example

{
    "id": "HELLO-01",
    "name": "Hello world",
    "mobile": true,
    "notes": "Initial import; first pass.",
    "syntax": 4,
    "version": 1.00,
    "style": {
        "replacements": [
            {
                "type": "color",
                "name": "Background Color",
                "find": "#FFFFFE"
            }
        ]
    },
    "elements": [
        {
            "id": "logo",
            "name": "Logo",
            "type": "image"
        }
    ],
    "variables": {}
}

Available Properties

id (string)
required
A unique identifier for the template.
Example(s): "123456", "LP-1234", "zBxVL", etc.
name (string)
required
The name of the template. This can be changed within the LeadPages interface, if needed.
Example(s): "Video Squeeze Page", Facebook Promo Page", etc.
mobile (boolean)
optional
Is the template mobile responsive?
Example(s): true, false (default)
notes (string)
required
A brief description of the template or a comment concerning any changes (for updates).
Example(s): "Initial creation!"
syntax (integer)
required
The syntax for the template. Currently the system only accepts a syntax value of 4.
Example(s): 4
version (float)
required
The current version number of the template. Each subsequent update to a template must increment the version number in order to upload properly.
Example(s): 1.0, 2.19
elements (array)
required
An array of objects that informs the LeadPages editor of elements that are editable. Requires at least one object to upload successfully.
Example(s): See each element for more information on the available object properties.
style (object)
required
An object containing an array of replacement objects (read: instructions).
Example(s): See the styles section for more information on the available properties of a replacement object.
variables (object)
optional
An object that connects an element with configuration data and any Javascript methods that might utilize the leadpages_input_data object.
Example(s): See the Facebook Share Button for an example.

Styles

A breakdown of the tools available for color customization (styles). The "style" property within meta/template.json currently supports only one property - "replacements". The "replacements" property expects an array of objects. Below is an example.

"Replacement" Objects

"style": {
    "replacements": [
        {
            "type": "color",
            "name": "Background Color",
            "find": "#FFFFFE"
        }
    ]
}

Available Properties

type (string)
required
The type of style component to render in the editor.
Example(s): "Color" or "Font".
name (string)
required
The label of the style to change under the "Styles" section of the editor.
Example(s): "Background Color", "Main Font Color", "Link Color".
find (string)
required
The value to "find" and subsequently replace. Note, because this operation is a find/replace, using a common color in css/style.css for many items will result in a larger-than-expected replacement.
Example(s): Setting "find" to "#CCCCCC" will change anything using that color, regardless of the cosmetic intent. It's strongly recommended to use slight variations for more granular control (i.e. "#CCCCCD", "#CCCCCB", etc.).

Elements

Containers

Containers are elements within a template that can be hidden. Technically speaking any element in a template that can be edited, can be hidden. The "container" type (meta/template.json) is for elements that only need this feature.

Container
index.html snippet
<div id="ex01-container" class="well">...</div>
meta/template.json snippet
"elements": [{
    "id": "ex01-container",
    "name": "Container",
    "type": "container"
}]

Text

The "text" type (meta/template.json) designates the enclosed text as editable. A limited number of formatting options are available for the enclosed text (i.e. bold, italic, underlined, foreground and background colors, links, and the font.)

index.html snippet
<p id="s01-lead">...</p>
meta/template.json snippet
"elements": [{
    "id": "ex02-lead",
    "name": "Lead",
    "type": "text"
}]

Images

The "image" type (meta/template.json) designates an image as swappable. The "max-width" property will provide advice when choosing an image but will not actually constrain the image dimensions. That behavior is best handled in the stylesheets.

index.html snippet
<img src="img/340x120.png" id="ex03-image" alt="" class="thumbnail" />
meta/template.json snippet
"elements": [{
    "id": "ex03-image",
    "name": "Example Image",
    "type": "image",
    "comment": "Choose an example.",
    "data": {
        "max-width": 340
    }
}]

Background Images

The "background-image" type (meta/template.json) is identical to the "image" type, minus the hover effect in the template editor. This subtle difference is very important for templates that utilize large images under other HTML elements.

index.html snippet
<div id="ex04-background-image">...</div>
meta/template.json snippet
"elements": [{
    "id": "ex04-background-image",
    "name": "Example 04 Bakground Image",
    "type": "background-image",
    "comment": "Choose a background image.",
    "data": {
        "max-width": 1140,
        "max-height": 500
    }
}]

Videos

The "video" type (meta/template.json) specifies a container within which the editor can embed a video. The editor will insert a placeholder during editing but including a mock image will help during the editing phase, prior to upload.

index.html snippet
<div id="ex05-video">...</div>
meta/template.json snippet
"elements": [{
    "type": "video",
    "id": "ex05-video",
    "name": "Example 05 Video"
}]

Text Links

The "link" type (meta/template.json) designates a hyperlink that can be edited. The "text" type should be used when links are amongst a larger body of text.

Example

index.html snippet
<a href="http://example.com" id="ex06-link">Example</a>
meta/template.json snippet
"elements": [{
    "type": "link",
    "id": "ex06-link",
    "name": "Example 6 Link"
}]

Image Links

The "image-link" type (meta/template.json) is a mash-up of the "image" and the "link" types. It allows image swapping and assigning a destination URL.

index.html snippet
<a href="http://example.com" id="ex06-image-link"><img src="img/340x120.png" alt="" class="thumbnail" /></a>
meta/template.json snippet
"elements": [{
    "type": "image-link",
    "id": "ex07-image-link",
    "name": "Example 7 Image Link",
    "data": {
        "max-width": 340
    }
}]

Embed

The "embed" type (meta/template.json) designates a container that can house embeddable content such as a Twitter widget, a Chatroll, SoundCloud track, or more.

Embed Something Here
index.html snippet
<div id="ex08-embed" class="well">Embed Something Here</div>
meta/template.json snippet
"elements": [{
    "type": "embed",
    "id": "ex08-embed",
    "name": "Example 8 Embed"
}]

Countdown Timer

The "countdown" type (meta/template.json) can be used to update sub-elements with the days, hours, minutes, or seconds remaining from now until a chosen date. The "days", "hours", "minutes", and "seconds" classes, are required. Without them the countdown timer will not function properly.

  • 6

    Days

  • 4

    Hours

  • 12

    Minutes

  • 44

    Seconds

index.html snippet
<div id="ex09-countdown">
    <ul class="countdown">
        <li>
            <div class="flap"><span class="days">6</span></div>
            <p class="timeRefDays" id="days-text">Days</p>
        </li>
        <li>
            <div class="flap">
                <span class="hours">4</span>
            </div>
            <p class="timeRefHours" id="hours-text">Hours</p>
        </li>
        <li>
            <div class="flap">
                <span class="minutes">12</span>
            </div>
            <p class="timeRefMinutes" id="minutes-text">Minutes</p>
        </li>
        <li>
            <div class="flap">
                <span class="seconds">44</span>
            </div>
            <p class="timeRefSeconds" id="seconds-text">Seconds</p>
        </li>
    </ul>
</div>
meta/template.json snippet
"elements": [{
    "type": "countdown",
    "id": "ex09-countdown",
    "name": "Example 9 Countdown"
}]

Facebook Share Button

The "dynamic" type (meta/template.json) can be used to transform an image link into a Facebook share button. An additional variable needs to be added to meta/template.json in order for the "dynamic" control to know where to pull data from. This data, updated within the editor, is ultimately passed to js/functions.json for use, say in a little popup window.

index.html snippet
<a href="" id="ex11-dynamic" class="pop-facebook">
    <img src="img/fb-blue-100.png" alt="" />
</a>
meta/template.json snippet
"elements": [{
    "type": "dynamic",
    "id": "ex11-dynamic",
    "name": "Example 11: Facebook Share Button",
    "data": {
        "variables": ["facebookurl"]
    }
},
"variables": {
    "facebookurl": {
        "variable": "facebookurl",
        "dafault": "",
        "name": "The URL to share on Facebook. Leave blank to share the current page."
    }
}]

LeadBoxes™

The "link" and "image-link" types (meta/template.json) can be pre-configured to trigger an opt-in form with one simple change. The addition of the "#/optin" value for the data.href property will default a "link" or "image-link" element to use the opt-in form instead of a anchor link. The default look-and-feel of an opt-in form can be further configured within the meta/form.html file.

Opt-in Button

index.html snippet
<a class="btn btn-warning btn-lg" id="ex15-link">Opt-in Button</a>
<a id="ex15-image-link"><img src="img/240x180.png" alt="" class="thumbnail" /></a>
meta/template.json snippet
"elements": [{
    "type": "link",
    "id": "ex15-link",
    "name": "Ex.15: Opt-in Link",
    "data": {
        "href": "#/optin"
    }
}]
"elements": [{
    "type": "image-link",
    "id": "ex15-image-link",
    "name": "Ex.15: Opt-in Image Link",
    "data": {
        "max-width": 240,
        "href": "#/optin"
    }
}]

Available Meta Properties

  • lp-popup-form-css: add form-specific CSS
  • lp-popup-form-title-html: set the default title
  • lp-popup-form-title-css: add title-specific CSS
